## ğŸ¯ EXECUTIVE SUMMARY

### What We're Building

A dual-interface web application that serves two audiences:

1. **CLIENT PORTAL**: Where Ian's coaching clients log in to:
   - View their enriched leads (scored and qualified)
   - Claim high-value leads they want to personally work
   - Add manual notes and track their own interactions
   - See lead status and campaign activity
   - Get notified of hot opportunities

2. **ADMIN DASHBOARD**: Where Ian and his team:
   - Control the entire SMS/WhatsApp system
   - Monitor all campaigns across all clients
   - Review conversations requiring human attention
   - Manage system settings and notifications
   - Analyze performance and ROI

### Core Value Proposition

**For Clients:**
"Log in, see your best leads scored and ready to work, claim the ones you want, and let automation handle the rest."

**For Ian:**
"One system to manage lead generation for all coaching clients, with full visibility and control."

## ğŸ‘¥ USER ROLES & PERMISSIONS

### Role 1: SUPER ADMIN (Ian)
**Can do everything:**
- View all clients and their leads
- Control all campaigns
- Access all system settings
- Manage user accounts
- See full analytics across all clients

### Role 2: ADMIN (Ian's Team)
**Can manage operations:**
- View all clients and leads
- Control campaigns
- Review conversation queue
- Configure notifications
- Cannot manage users or billing

### Role 3: CLIENT (Coaching Customer)
**Can manage their own leads:**
- View only their leads (filtered by Client ID)
- Claim leads for personal outreach
- Add notes to their leads
- See campaign status for their leads
- Configure their own notifications
- Cannot see other clients' data
- Cannot control system settings

### Security Features

1. **Password Requirements**
   - Minimum 12 characters
   - Must include uppercase, lowercase, number, symbol
   - Cannot reuse last 5 passwords
   - Expires every 90 days (optional)

2. **Two-Factor Authentication** (Optional)
   - SMS-based 2FA
   - Authenticator app (Google Authenticator)
   - Backup codes provided

3. **Role-Based Access Control**
   ```javascript
   Permissions by Role:
   
   SUPER_ADMIN:
     - clients: [read, write, delete]
     - leads: [read, write, delete] (all clients)
     - campaigns: [read, write, delete, control]
     - system: [read, write, configure]
     - users: [read, write, delete]
   
   ADMIN:
     - clients: [read]
     - leads: [read, write] (all clients)
     - campaigns: [read, write, control]
     - system: [read]
     - users: [read]
   
   CLIENT:
     - clients: [read] (own only)
     - leads: [read, write, claim] (own only)
     - campaigns: [read] (own only)
     - system: [read] (limited)
     - users: [read] (own team only)
   ```

4. **Data Isolation**
   - Clients can ONLY see their own leads
   - Database queries auto-filtered by Client ID
   - API endpoints validate client ownership
   - Cross-client data leakage impossible

5. **Audit Logging**
   - Every action logged with:
     * Who (user ID + name)
     * What (action type)
     * When (timestamp)
     * Where (IP address)
     * Result (success/failure)

### Airtable â†” Web App Sync

**Challenge**: Airtable is the source of truth for leads, but web app needs fast read access.

**Solution**: Hybrid approach

#### 1. Initial Sync (When Lead Created in Airtable)
```
Airtable (via automation) â†’ Webhook â†’ n8n â†’ Web App API
                                            â†“
                                      Create lead in DB
```

#### 2. Real-Time Updates (When Lead Changes)
```
Airtable automation detects change â†’ Webhook â†’ n8n â†’ Web App API
                                                      â†“
                                                 Update lead in DB
```

#### 3. Web App Changes (When Client Claims Lead)
```
Client clicks "Claim" â†’ Web App API â†’ Update local DB
                                    â†’ Update Airtable via MCP
                                    â†’ Pause campaign in n8n
```

#### 4. Periodic Full Sync (Safety Net)
```
Every 1 hour: n8n workflow runs
  â†“
Fetch all leads from Airtable
  â†“
Compare with web app DB
  â†“
Reconcile any differences
  â†“
Log discrepancies for review
```

### Performance Optimization

**Read Operations** (Most Common):
- Client views lead list â†’ Read from web app DB (fast!)
- No Airtable API call needed
- Sub-100ms response time

**Write Operations** (Less Frequent):
- Client claims lead â†’ Update both DB and Airtable
- Transaction-safe: If either fails, rollback
- Max 500ms response time

**Real-Time Events**:
- New SMS reply comes in â†’ n8n processes â†’ Updates Airtable â†’ Webhook to web app
- Web app shows notification immediately
- SSE (Server-Sent Events) pushes to connected clients

```javascript
// Calculate ICP score (0-100)
function calculateICPScore(lead) {
  let score = 0;
  
  // Title scoring (30 points max)
  if (lead.title.match(/VP|Vice President/i)) score += 30;
  else if (lead.title.match(/Director/i)) score += 25;
  else if (lead.title.match(/Manager/i)) score += 15;
  
  // Company size (20 points max)
  if (lead.companySize >= 500) score += 20;
  else if (lead.companySize >= 100) score += 15;
  else if (lead.companySize >= 50) score += 10;
  
  // Industry fit (25 points max)
  const targetIndustries = ['Tech', 'SaaS', 'Software'];
  if (targetIndustries.includes(lead.industry)) score += 25;
  
  // Location (10 points max)
  const targetLocations = ['SF', 'NYC', 'Austin', 'Seattle'];
  if (targetLocations.some(loc => lead.location.includes(loc))) {
    score += 10;
  }
  
  // Engagement history (15 points max)
  if (lead.attendedWebinar) score += 15;
  else if (lead.downloadedResource) score += 10;
  
  return Math.min(score, 100);
}
```
